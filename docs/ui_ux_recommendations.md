# UI/UX рекомендации

## Наблюдения
- В AI Review привязка к предметам/локациям требует ручного ввода ID → высокая когнитивная нагрузка и риск ошибок.
- Ошибки сети/AI скрыты: accept/reject не показывают тост/снейкбар об успехе/провале.
- История загрузок отображает статусы, но нет визуального отличия pending/in_progress/failed, нет фильтрации.
- Bottom bar использует одинаковый стиль иконок, но экран поиска спрятан (нет явного входа).
- Шаринг из других приложений работает, но нет явного прогресса и подтверждения успешного импорта.

## Рекомендации и быстрые победы
1) **Упростить привязку AI**: добавить поиск по предметам/локациям (typeahead + недавние) и кнопки "Создать новый"/"Исправить локацию", вместо ввода ID.
2) **Обратная связь**: после accept/reject/update показывать снейкбар с результатом, ошибки сети — отдельным сообщением; лоадер/disabled на кнопках.
3) **Статусы и фильтры**: в истории загрузок подсветить статусы цветом (success/pending/failed), дать фильтр "Только ошибки/ожидание".
4) **AI превью**: отображать миниатюру bbox/label в карточке детекции; по клику — полноразмерное изображение.
5) **Шаринг**: показывать прогресс импорта (URL/файл), итоговый результат (создан item #123), кнопку "Открыть".
6) **Пустые состояния**: для Queue/History/Search добавить пустые экраны с CTA (например, "Загрузите фото, чтобы увидеть детекции").

## Дизайн-направление
- Сохранить лаконичность: нейтральный фон, чёткие карточки, акцентными сделать состояния (зелёный успех, янтарный pending, красный failed).
- Использовать крупные tap-мишени (48dp) для действий AI и загрузок.
- Добавить анимацию появления карточек в AI Review/History (fade + slide) для лучшей ориентации.
