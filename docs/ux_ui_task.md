# Задачи по UX/UI

Цель: ускорить добавление и поиск вещей, сделать структуру локаций и AI-подтверждения понятными и быстрыми.

Ключевые задачи
- Быстрое добавление: минимальный ввод (название + локация + медиа), черновики, «сохранить и следующая».
- Локации: иерархия дом → квартира → комната → шкаф → ящик → коробка; создание/переименование/перемещение/удаление.
- Массовое добавление: серия фото/видео из одной локации/коробки с прогрессом и автоматической привязкой.
- Медиа: добавление фото/видео к вещи, просмотр в карточке, отправка в AI.
- AI: отдельный инбокс предложений, быстрые действия Принять/Отклонить/Связать/Создать, лог решений.
- Навигация: bottom bar с вкладками Вещи/Локации/Добавить/AI/Настройки, расширяемость под фильтры, теги, совместный доступ.

User flows (минимально)
- Добавление вещи: открыть «Добавить» → ввести название → выбрать/создать локацию → добавить фото → сохранить (или «сохранить и следующая»).
- Изменение локации: в карточке вещи «Переместить» → выбрать новую локацию → подтвердить.
- Работа с медиа: в карточке вещи добавить фото/видео → просмотр в карусели → открыть полноэкранно.
- AI-подтверждение: открыть AI-инбокс → просмотреть превью/кандидатов → Принять/Отклонить/Связать/Создать → лог записывается.

Критерии готовности
- Вещь добавляется за ≤20 секунд в 2–3 шага.
- Структура локаций видна и редактируема за 1–2 действия.
- AI-инбокс позволяет подтвердить/отклонить за 1–2 тапа, лог сохраняется.
- Все ключевые действия логируются для аналитики UX.
